## 🚀 3374. First Letter Capitalization II

### 💡 Idea

The goal is to transform the `content_text` column by following these rules:

- Capitalize the **first letter** of every word.
- Convert all remaining letters to lowercase.
- If a word contains **exactly one hyphen (`-`)**, capitalize the first letter of both parts.
- Preserve all other spaces and special characters.

Since MySQL does not provide a built-in `INITCAP()` function, we solve the problem by processing the string **one character at a time** using **Recursive CTEs**.

---

## 💻 Solution

```sql
-- Split every string into one character per row
WITH RECURSIVE cte AS (
    SELECT content_id, content_text, LENGTH(content_text) AS len
    FROM user_content
),

chars AS (
    -- Anchor member
    SELECT content_id, 1 AS pos,
    LOWER(SUBSTRING(content_text,1,1)) AS ch,
    LOWER(content_text) AS txt,
    len, 0 AS word_idx
    FROM cte

    UNION ALL

    -- Recursive member
    SELECT content_id, pos + 1,
    LOWER(SUBSTRING(txt,pos+1,1)),
    txt, len,
    word_idx + CASE WHEN ch = ' ' THEN 1 ELSE 0 END
    FROM chars
    WHERE pos < len
),

word_info AS (
    SELECT content_id, word_idx,
    SUM(CASE WHEN ch='-' THEN 1 ELSE 0 END) AS hyphen_count,
    MIN(pos) AS word_start_pos,
    MIN(CASE WHEN ch='-' THEN pos END) AS first_hyphen_pos
    FROM chars
    GROUP BY content_id, word_idx
)

SELECT c.content_id,
MAX(u.content_text) AS original_text,
GROUP_CONCAT(
            CASE

            -- First character of sentence
            WHEN c.pos = 1 THEN UPPER(c.ch)

            -- First character after space
            WHEN SUBSTRING(c.txt,c.pos-1,1)=' ' THEN UPPER(c.ch)

            -- First character after hyphen
            WHEN SUBSTRING(c.txt,c.pos-1,1)='-'
            AND wi.hyphen_count=1
            AND wi.first_hyphen_pos<>wi.word_start_pos
            THEN UPPER(c.ch)

            ELSE c.ch
        END
        ORDER BY c.pos
        SEPARATOR ''
    ) AS converted_text
FROM chars c
JOIN user_content u
ON c.content_id=u.content_id
JOIN word_info wi
ON c.content_id=wi.content_id
AND c.word_idx=wi.word_idx
GROUP BY c.content_id
ORDER BY c.content_id;
```

---

## 🛠️ How It Works

### **Step 1: Calculate the length of every string**

```sql
cte
```

The first CTE simply stores:

- `content_id`
- `content_text`
- total string length

Example:

| content_id | content_text | len |
|------------|--------------|----:|
|1|hello world|11|

The string length is used to stop the recursive process.

---

### **Step 2: Split the string into individual characters**

The recursive CTE `chars` processes **one character at a time**.

For example,

```
Hello World
```

becomes

| pos | ch |
|----:|----|
|1|h|
|2|e|
|3|l|
|4|l|
|5|o|
|6| |
|7|w|
|8|o|
|9|r|
|10|l|
|11|d|

Every character becomes its own row.

---

### **Step 3: Track word numbers**

```sql
word_idx
```

Every time a space is encountered,

```sql
CASE
WHEN ch=' '
THEN 1
ELSE 0
END
```

increments the word number.

For example,

```
Hello SQL World
```

becomes

| Character | word_idx |
|-----------|---------:|
|Hello|0|
|SQL|1|
|World|2|

This allows us to analyze each word independently.

---

### **Step 4: Gather information about each word**

The `word_info` CTE calculates:

- number of hyphens
- starting position of the word
- first hyphen position

Example

```
top-rated
```

produces

| hyphen_count | word_start_pos | first_hyphen_pos |
|--------------|---------------:|-----------------:|
|1|1|4|

While

```
---hello
```

does not satisfy the required condition.

---

### **Step 5: Decide which letters should be capitalized**

The `CASE` statement determines whether the current character should become uppercase.

#### First character of the sentence

```sql
WHEN c.pos = 1
```

Example

```
hello

↓

Hello
```

---

#### First character after a space

```sql
WHEN SUBSTRING(c.txt,c.pos-1,1)=' '
```

Example

```
hello world

↓

Hello World
```

---

#### First character after a hyphen

```sql
WHEN SUBSTRING(c.txt,c.pos-1,1)='-'
```

This condition is only applied when:

- the word contains exactly one hyphen
- the hyphen is not the first character of the word

Example

```
top-rated

↓

Top-Rated
```

---

#### Otherwise

Every other character remains lowercase.

---

### **Step 6: Rebuild the sentence**

During recursion, each character is stored separately.

`GROUP_CONCAT()` joins them back together in the correct order.

Example

| pos | character |
|----:|-----------|
|1|H|
|2|e|
|3|l|
|4|l|
|5|o|

becomes

```
Hello
```

---

### ⏱️ Complexity

**Time Complexity:** `O(n × m)`

- `n` = number of rows
- `m` = average length of each string

Each character is processed exactly once.

**Space Complexity:** `O(n × m)`

The recursive CTE stores one row for every character in every string.

### 💡 Interview Tip: PostgreSQL Approach

Unlike MySQL, PostgreSQL provides built-in functions that make string capitalization much easier.

PostgreSQL has:

```sql
INITCAP()
```

which converts the first letter of every word to uppercase and the remaining letters to lowercase.

For example:

```sql
SELECT INITCAP('hello world');
```

Output:

```
Hello World
```

---

A basic solution would be:

```sql
SELECT
    content_id,
    content_text,
    INITCAP(content_text) AS converted_text
FROM user_content;
```

However, `INITCAP()` does not correctly handle hyphenated words in all cases.

Example:

```text
top-rated
```

may become:

```text
Top-Rated
```

or behave differently depending on the PostgreSQL version and punctuation handling.

---

For complete control, PostgreSQL can combine `LOWER()`, `REGEXP_REPLACE()`, and regex patterns:

```sql
SELECT
    content_id,
    content_text,
    REGEXP_REPLACE(
        INITCAP(LOWER(content_text)),
        '([^-])-([a-z])',
        '\1-\U\2',
        'g'
    ) AS converted_text
FROM user_content;
```

### Why PostgreSQL is easier:

| Feature | MySQL | PostgreSQL |
|---|---|---|
| `INITCAP()` | ❌ Not available | ✅ Available |
| Regex replacement | Limited | Powerful |
| String processing | More manual | Built-in support |

Therefore, in PostgreSQL this problem can be solved in a few lines, while MySQL requires recursive CTEs and character-level processing.

For data engineering and analytics roles, knowing these database-specific functions is valuable because the same transformation can require very different approaches depending on the SQL engine.
