## 🚀 601. Human Traffic of Stadium

### 💡 Idea

The goal is to find all records that belong to **three or more consecutive days** where the number of visitors is **at least 100**.

There are two common approaches:

- **Approach 1:** Use a **self join** to compare consecutive records.
- **Approach 2:** Use the `LAG()` and `LEAD()` window functions to check neighboring records.

---

## 💻 Approach 1: Self Join

```sql
SELECT DISTINCT s1.*
FROM Stadium s1, Stadium s2, Stadium s3
WHERE s1.people >= 100 AND s2.people >= 100 AND s3.people >= 100
  AND (
        (s1.id = s2.id - 1 AND s2.id = s3.id - 1) # s1 s2 s3 (s1 = first consecutive number)
     OR (s2.id = s1.id - 1 AND s1.id = s3.id - 1) # s2 s1 s3 (s1 = second consecutive number)
     OR (s2.id = s3.id - 1 AND s3.id = s1.id - 1) # s2 s3 s1 (s1 = third consecutive number)
  )
ORDER BY id;

```

### 🛠️ How It Works

1. Treat the `Stadium` table as three separate tables (`s1`, `s2`, and `s3`).
2. Keep only rows where all three records have `people >= 100`.
3. Check whether the three IDs are consecutive.
4. Consider all three positions of `s1`:
   - First in the sequence.
   - Middle of the sequence.
   - Last in the sequence.
5. Use `DISTINCT` to remove duplicate rows.
6. Return the result ordered by `id`.

---

## 💻 Approach 2: `LAG()` & `LEAD()`

```sql
SELECT
    id,
    visit_date,
    people
FROM (
    SELECT
        *,
        LAG(id, 1) OVER (ORDER BY id) AS prev_id,
        LAG(id, 2) OVER (ORDER BY id) AS prev2_id,
        LEAD(id, 1) OVER (ORDER BY id) AS next_id,
        LEAD(id, 2) OVER (ORDER BY id) AS next2_id
    FROM Stadium
    WHERE people >= 100
) AS t
WHERE
      (id = prev_id + 1 AND id = prev2_id + 2)
   OR (id = prev_id + 1 AND id = next_id - 1)
   OR (id = next_id - 1 AND id = next2_id - 2)
ORDER BY id;
```

### 🛠️ How It Works

1. Filter the rows where `people >= 100`.
2. Use `LAG()` to retrieve the previous one and two IDs.
3. Use `LEAD()` to retrieve the next one and two IDs.
4. Check whether the current row is:
   - The last row of a sequence.
   - The middle row of a sequence.
   - The first row of a sequence.
5. Return all qualifying rows ordered by `id`.

---

### 📌 Which Approach Should You Use?

- ⭐ **Approach 2 (`LAG()` / `LEAD()`):** Recommended because it is cleaner, easier to understand, and avoids multiple self joins.
- ✅ **Approach 1 (Self Join):** Useful when window functions are unavailable or when practicing self joins.

---

### ⏱️ Complexity

| Approach | Time | Space |
|----------|------|-------|
| Self Join | `O(n²)` | `O(1)` *(excluding the output result set)* |
| `LAG()` / `LEAD()` | `O(n log n)` | `O(n)` |
