## 🚀 1873. Calculate Special Bonus

### 💡 Idea

The goal is to calculate the bonus for each employee based on the following rules:

- The employee's **ID must be odd**.
- The employee's **name must not start with 'M'**.

If both conditions are satisfied, the bonus is equal to the employee's salary. Otherwise, the bonus is `0`.

We can solve this using either `LIKE` or `REGEXP` to check the employee's name.

---

## 💻 Approach 1: Using `LIKE`

```sql
SELECT employee_id,
CASE WHEN employee_id % 2 = 1
AND name NOT LIKE 'M%' THEN salary
ELSE 0
END AS bonus
FROM Employees
GROUP BY employee_id
ORDER BY employee_id;
```

### 🛠️ How It Works

1. Check if the employee ID is odd using `employee_id % 2 = 1`.
2. Use `NOT LIKE 'M%'` to ensure the name does not start with `'M'`.
3. If both conditions are true, return the employee's salary.
4. Otherwise, return `0`.
5. Sort the result by `employee_id`.

---

## 💻 Approach 2: Using `REGEXP`

```sql
SELECT employee_id,
CASE WHEN MOD(employee_id, 2) != 0 AND name NOT REGEXP '^[M]' THEN salary
ELSE 0
END AS bonus
FROM Employees
GROUP BY employee_id
ORDER BY employee_id;
```

### 🛠️ How It Works

1. `MOD(employee_id, 2) != 0` checks whether the employee ID is odd.
2. `NOT REGEXP '^[M]'` ensures the name does **not** begin with the letter `M`.
3. The `CASE` expression returns:
   - `salary` if both conditions are satisfied.
   - `0` otherwise.
4. The result is ordered by `employee_id`.

---

### ⏱️ Complexity

| Approach | Time | Space |
|----------|------|-------|
| `LIKE` | `O(n)` | `O(1)` |
| `REGEXP` | `O(n × m)` | `O(1)` |

*(`m` = average length of the employee name)*
