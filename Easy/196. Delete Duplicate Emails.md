## 🚀 196. Delete Duplicate Emails

### 💡 Idea

To remove duplicate emails, we need to keep **only the record with the smallest `id`** for each email and delete the rest.

This can be achieved using a **self join**, where the `Person` table is joined with itself. For duplicate emails, the row with the larger `id` is identified and deleted.

---

### 💻 Solution

```sql
DELETE p1
FROM Person p1
INNER JOIN Person p2
ON p1.email = p2.email
WHERE p1.id > p2.id;
```

---

### 🛠️ How It Works

1. Perform a **self join** on the `Person` table using the `email` column.
2. Match records that have the same email address.
3. Compare their `id` values.
4. If `p1.id` is greater than `p2.id`, `p1` is the duplicate record.
5. Delete the duplicate record while keeping the one with the smallest `id`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n²)` *(depends on the self join and indexing on `email`)*

**Space Complexity:** `O(1)` *(excluding the execution overhead)*
