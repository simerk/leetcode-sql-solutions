## 🚀 607. Sales Person

### 💡 Idea

Instead of joining all three tables, we only need to identify the `sales_id` values that have placed an order for the company **'Red'**. Then, we simply exclude those `sales_id`s from the **SalesPerson** table using `NOT IN`.

This approach is shorter, more efficient, and avoids an unnecessary join with the `SalesPerson` table inside the subquery.

---

### 💻 Solution

```sql
SELECT s.name
FROM SalesPerson s
WHERE s.sales_id NOT IN (
    SELECT o.sales_id
    FROM Orders o
    INNER JOIN Company c
    ON c.com_id = o.com_id
    WHERE c.name = 'Red'
);
```

---

### 🛠️ How It Works

1. Join the **Orders** and **Company** tables.
2. Filter the orders where the company name is **'Red'**.
3. The subquery returns the `sales_id` of every salesperson who has sold to **Red**.
4. The outer query selects all salespersons whose `sales_id` is **not** in that list.
5. The final result contains only the salespersons who have never sold to **Red**.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(depends on table size and indexing)*

**Space Complexity:** `O(n)` *(stores the list of `sales_id`s returned by the subquery)*
