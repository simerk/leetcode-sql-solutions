## 🚀 184. Department Highest Salary

### 💡 Idea

The goal is to find the employee(s) with the **highest salary in each department**.

We can solve this problem in two ways:

- **Approach 1:** Use a correlated subquery to compare each employee's salary with the maximum salary in their department.
- **Approach 2:** Use the `DENSE_RANK()` window function to rank employees by salary within each department and select those with rank `1`.

---

## 💻 Approach 1: Correlated Subquery

```sql
SELECT
    d.name AS Department,
    e.name AS Employee,
    e.salary AS Salary
FROM Employee e
INNER JOIN Department d
ON e.departmentId = d.id
WHERE e.salary = (
    SELECT MAX(salary)
    FROM Employee
    WHERE departmentId = e.departmentId
);
```

### 🛠️ How It Works

1. Join the `Employee` and `Department` tables.
2. For each employee, find the maximum salary in that employee's department using a correlated subquery.
3. Compare the employee's salary with the maximum salary.
4. Return employees whose salary matches the maximum salary in their department.

---

## 💻 Approach 2: `DENSE_RANK()` Window Function

```sql
SELECT
    Department,
    Employee,
    Salary
FROM (
    SELECT
        d.name AS Department,
        e.name AS Employee,
        e.salary AS Salary,
        DENSE_RANK() OVER (
            PARTITION BY e.departmentId
            ORDER BY e.salary DESC
        ) AS rnk
    FROM Employee e
    INNER JOIN Department d
    ON e.departmentId = d.id
) AS ranked
WHERE rnk = 1;
```

### 🛠️ How It Works

1. Join the `Employee` and `Department` tables.
2. Partition the data by `departmentId`.
3. Rank employees within each department based on salary in descending order using `DENSE_RANK()`.
4. Employees with the highest salary receive a rank of `1`.
5. Filter the result to return only employees with rank `1`.

---

### 📌 Which Approach Should You Use?

- ✅ **Approach 1:** Works in databases that don't support window functions and is easy to understand.
- ⭐ **Approach 2:** Recommended when window functions are available because it's more scalable, easier to extend (e.g., top 3 salaries), and avoids executing a correlated subquery for each row.

---

### ⏱️ Complexity

| Approach | Time | Space |
|----------|------|-------|
| Correlated Subquery | `O(n²)` | `O(1)` |
| `DENSE_RANK()` | `O(n log n)` | `O(n)` |
