## 🚀 3220. Odd and Even Transactions

### 💡 Idea

The goal is to calculate, for each transaction date:

- The total amount of **odd-valued transactions**.
- The total amount of **even-valued transactions**.

We use **conditional aggregation** with `CASE` expressions to separately sum the transaction amounts based on whether the **amount** is odd or even.

---

## 💻 Solution

```sql
SELECT transaction_date,
SUM(CASE WHEN amount % 2 != 0 THEN amount ELSE 0 END) AS odd_sum,
SUM(CASE WHEN amount % 2 = 0 THEN amount ELSE 0 END) AS even_sum
FROM Transactions
GROUP BY transaction_date
ORDER BY transaction_date;
```

---

### 🛠️ How It Works

1. Group all transactions by `transaction_date`.
2. Check whether the **transaction amount** is odd using `amount % 2 != 0`.
3. Sum only the odd amounts into `odd_sum`.
4. Check whether the **transaction amount** is even using `amount % 2 = 0`.
5. Sum only the even amounts into `even_sum`.
6. Return the result ordered by `transaction_date`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(single scan followed by grouping)*

**Space Complexity:** `O(n)` *(stores grouped dates)*
