## 🚀 1164. Product Price at a Given Date

### 💡 Idea

The goal is to find the price of every product on **2019-08-16**.

- If a product has one or more price changes **on or before** `2019-08-16`, its price is the **latest** price before or on that date.
- If a product has **no price changes** before that date, its price is assumed to be **10**.

We use the `ROW_NUMBER()` window function to identify the latest price change for each product before the given date, then assign the default price to products with no qualifying records.

---

## 💻 Approach 1: `ROW_NUMBER()` (Recommended)

```sql
SELECT p.product_id, IFNULL(t.new_price, 10) AS price
FROM (
    SELECT DISTINCT product_id FROM Products
) p
LEFT JOIN (
    SELECT product_id, new_price
    FROM (
        SELECT *,
               ROW_NUMBER() OVER (
                   PARTITION BY product_id
                   ORDER BY change_date DESC
               ) AS rn
        FROM Products
        WHERE change_date <= '2019-08-16'
    ) p2
    WHERE rn = 1
) t
ON p.product_id = t.product_id;
```

### 🛠️ How It Works

1. Retrieve all distinct `product_id`s from the `Products` table.
2. Filter price changes that occurred on or before **2019-08-16**.
3. Use `ROW_NUMBER()` to rank price changes for each product by `change_date` in descending order.
4. Keep only the latest price change (`rn = 1`) for each product.
5. Perform a `LEFT JOIN` with all products.
6. Use `IFNULL()` to assign a default price of **10** to products with no price changes before the given date.

---

## 💻 Approach 2: `MAX()` + `INNER JOIN`

```sql
SELECT p.product_id, IFNULL(t.new_price, 10) AS price
FROM (
    SELECT DISTINCT product_id FROM Products
) p
LEFT JOIN (
    SELECT p1.product_id, p1.new_price
    FROM Products p1
    INNER JOIN (
        SELECT
            product_id,
            MAX(change_date) AS change_date
        FROM Products
        WHERE change_date <= '2019-08-16'
        GROUP BY product_id
    ) p2
    ON p1.product_id = p2.product_id
    AND p1.change_date = p2.change_date
) t
ON p.product_id = t.product_id;
```

### 🛠️ How It Works

1. Find the latest `change_date` for each product before or on **2019-08-16**.
2. Join the result back to the `Products` table to retrieve the corresponding `new_price`.
3. `LEFT JOIN` the result with the list of all products.
4. Use `IFNULL()` to assign the default price of **10** where no previous price exists.

---

### 📌 Which Approach Should You Use?

- ⭐ **Approach 1 (`ROW_NUMBER()`)** is the preferred solution. It is concise, easy to read, and scales well for problems involving the latest or top record within each group.
- ✅ **Approach 2 (`MAX()` + `JOIN`)** is another excellent solution and is especially useful in SQL dialects that do not support window functions.

---

### ⏱️ Complexity

| Approach | Time | Space |
|----------|------|-------|
| `ROW_NUMBER()` | `O(n log n)` | `O(n)` |
| `MAX()` + `JOIN` | `O(n)` | `O(n)` |
